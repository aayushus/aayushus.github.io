The AI-First
SDLC Playbook

A practical operating model for engineering teams. Six named AI agents handle implementation, code review, QA, security, releases, and maintenance — autonomously, overnight, on a schedule. Humans write requirements and approve the critical decisions.

Jira ticket → AI implements → AI reviews → QA agent tests → human approves → release agent deploys

⚖️
Humans stay in control — agents stop, ask, and wait
Agents never merge without human approval. Branch protections apply equally to agents and engineers.
If acceptance criteria are missing or ambiguous, Hermes posts a Jira comment asking for clarification and stops.
No auto-merge for auth, payments, or database changes — Themis flags these for mandatory human review.
Override any running agent from Teams at any time. Send EXIT to end a session immediately.
