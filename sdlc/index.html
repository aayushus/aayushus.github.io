The AI-First
SDLC Playbook

This playbook is a complete, AI-first operating model for the software development lifecycle. Seven named Devin agents — Builder, Reviewer, Tester, Fixer, Shipper, Maintainer, and InfoSec — run headlessly on a schedule across Jira, GitHub, Microsoft Teams, and Confluence, carrying every Jira Story from a Ready-for-Work ticket through implementation, code review, QA, autonomous bug-fixing, security review, release, and weekly maintenance. Humans stay in control of the decisions that matter — writing requirements and approving merges, sensitive changes, and releases — while the agents handle the routine work in between, overnight and without being asked.

The problem & the solution
The pain point — today's manual SDLC
Engineers spend their time on toil, not judgment
Code review is the bottleneck — pull requests wait 24–48 hours for a human.
QA is manual and slow; test coverage is inconsistent and unenforced.
Jira ↔ GitHub links, release notes, and runbooks are hand-maintained — and drift out of date.
Dependency updates, dead code, and CVEs get deferred until something breaks.
Context lives in people's heads, not the ticket — so work stalls at every handoff.
The solution — an AI-first agent team
Seven named agents own the routine work end-to-end
Builder picks up Ready-for-Work tickets overnight and opens complete pull requests.
Reviewer and InfoSec pre-review every PR in minutes; Tester enforces coverage and catches regressions nightly.
Fixer closes defects autonomously; Shipper compiles changelogs and tags every release.
Maintainer keeps dependencies, dead code, and CVEs in check every week.
All config-driven on Jira, GitHub, Teams & Confluence — humans approve only the decisions that matter.

Seven named agents across the lifecycle
Builder
Development Agent
Picks up Jira tickets, implements end-to-end, opens pull requests with full descriptions.
Daily 9am · @Devin !implement
Reviewer
Code Review Agent
Reviews every pull request for bugs, security, and coverage before a human touches it.
Auto on every PR · @Devin !review
Tester
QA & Testing Agent
Runs full regression nightly, files Defects for failures, enforces coverage thresholds.
Nightly 2am · @Devin !test
Fixer
Bug Fixer Agent
Picks up Ready for Work Defects labelled JAI_Agent, reproduces, fixes the root cause, and opens a PR. Closes the loop with Tester.
Daily 7am · @Devin !fix
Shipper
Release Agent
Compiles changelog, tags release in GitHub, updates Confluence, closes Jira version.
Friday 9am · @Devin !release
Maintainer
Maintenance Agent
Scans for outdated dependencies, dead code, and security CVEs. Opens fix PRs.
Monday 7am · @Devin !maintenance
InfoSec
Security Agent
Deep security review on every PR touching auth, payments, or data access paths.
On sensitive PRs · @Devin !security-review
⚖️
Humans stay in control — agents stop, ask, and wait
Agents never merge without human approval. Branch protections apply equally to agents and engineers.
If acceptance criteria are missing or ambiguous, Builder posts a Jira comment asking for clarification and stops.
No auto-merge for auth, payments, or database changes — InfoSec flags these for mandatory human review.
2 - HIGH and 3 - MEDIUM Defect fixes run autonomously through Reviewer auto-review — but 1 - URGENT fix PRs always require human approval before they merge.
Override any running agent from Teams at any time. Send EXIT to end a session immediately.
